# 신고해줘 - 프로덕트 디자인 스펙

## 개요

**프로덕트명:** 신고해줘
**한 줄 설명:** 직원 정보만 연동하면 4대보험 신고서를 자동 작성하고 관할 지사에 FAX를 발송하는 서비스
**타겟 사용자:** ERP/EDI 없이 4대보험을 직접 신고하는 HR 담당자 (소규모 사업장 5~50인)
**기술 스택:** Next.js + Tailwind CSS + Vercel + 팝빌 FAX API

## 문제 정의

소규모 회사의 HR 담당자는 직원 입사/퇴사 시 4대보험 신고를 수동으로 처리해야 한다:

**방법 A (FAX 신고):**
1. 4대보험 정보연계센터에서 서식 다운로드
2. 직원 정보를 보며 서식 수기 작성 (보험별로 필드가 다름)
3. 관할 지사 FAX 번호 검색
4. FAX 발송

**방법 B (온라인 신고):**
1. 4대보험 정보연계센터 접속 → 공동인증서 로그인
2. 필드마다 구성원 정보를 수작업 입력 (파편화된 정보 수집 필요)
3. 사유에 따라 해당 센터에서 처리 불가능하거나, 별도 ERP/EDI 시스템에 접속해야 하는 경우 발생

입사자 1명 기준 약 20~30분 소요. 보험별 입력 필드 차이로 실수 발생 위험. 어떤 방법을 쓰든 파편화된 직원 정보를 모아서 반복 입력하는 수작업이 핵심 병목.

### 기존 대안과 한계

| 대안 | 한계 |
|------|------|
| 세무사/노무사 대행 | 월 10~30만원 비용 |
| ERP (더존, 위하고) | 연 100만원+, 소규모에 과한 솔루션 |
| HR SaaS (Flex 등) | 서류 생성은 되나 FAX 발송은 수동, 월 인당 3,500~5,000원 |
| 직접 수동 처리 | 시간 소모, 실수 위험 |

**시장 빈틈:** "직원 정보 → 서식 자동작성 → FAX 자동발송"을 원스톱으로 제공하는 독립 서비스는 현재 없음.

## 핵심 플로우

```
1. 사업장 정보 설정 (최초 1회, localStorage 저장)
   - 사업장관리번호, 회사명, 소재지, 대표자명, 전화번호
   - 소재지 기반으로 관할지사 자동 매핑

2. 직원 정보 입력 (수동/CSV)
   → 신고 유형 선택 (입사 취득 / 퇴사 상실)
   → 서식 PDF 자동 생성 (브라우저에서 처리, 원본 서식 PDF 위에 텍스트 배치)
   → 발송 방식 선택:
       A. 원클릭 FAX 발송 (팝빌 API, 건당 50원)
       B. PDF 다운로드 + 관할지사 FAX번호 안내 (무료)
```

## 아키텍처

```
[사용자 브라우저]
     │
     ▼
[Next.js 앱 (Vercel)]
     │
     ├── / ─────────────── 랜딩 페이지
     ├── /app ────────────── 대시보드 (신고 이력)
     ├── /app/report ─────── 신고서 작성 플로우
     │     │
     │     ├── 1. 직원 정보 입력 (수동 / CSV 업로드)
     │     ├── 2. 신고 유형 선택 (입사취득 / 퇴사상실)
     │     ├── 3. 서식 미리보기 & PDF 생성 (브라우저)
     │     └── 4. 발송 선택
     │           ├── A. 원클릭 발송 → [팝빌 FAX API]
     │           └── B. 내 복합기 → PDF 다운 + 지사 FAX번호 안내
     │
     └── /api ────────────── API Routes
           ├── /api/send-fax ────── 팝빌 FAX 발송
           └── /api/offices ─────── 관할지사 조회
```

### 핵심 모듈

| 모듈 | 역할 | 기술 |
|------|------|------|
| 서식 엔진 | 직원 정보 → 4대보험 서식 PDF 생성 | pdf-lib (브라우저에서 실행). 원본 서식 PDF를 템플릿으로 로드 → 좌표 기반으로 텍스트 배치하여 필드 채움 |
| 지사 매핑 | 사업장 주소 → 관할 지사 + FAX번호 | JSON DB (4insure.or.kr 기반 수집) |
| FAX 발송 | PDF → 팝빌 API → 관할지사 FAX | 팝빌 Node.js SDK (서버사이드) |

### 데이터 처리 & 보안

- **개인정보 서버 미보관:** 직원 정보 입력과 PDF 생성은 브라우저에서 처리
- **FAX 발송 시:** PDF를 서버로 전송 → 팝빌 API 호출 → 즉시 삭제 (저장 안 함)
- **브라우저 저장:** localStorage로 입력 데이터 임시 보관 (사용자 디바이스에만 존재)

## MVP 범위 (9일, ~4/10)

### 포함

- **서식 2종:** 자격취득 신고서(입사), 자격상실 신고서(퇴사)
- **직원 정보 입력:** 수동 입력 폼 + CSV 업로드
- **PDF 자동 생성:** 브라우저 기반 (pdf-lib)
- **관할지사 매핑:** 전국 시/군/구 → 4대보험 기관별 지사 + FAX번호 (JSON)
- **발송 옵션 A:** 팝빌 FAX API 연동 (테스트 환경)
- **발송 옵션 B:** PDF 다운로드 + 관할지사 FAX번호 표시
- **신고 이력:** 브라우저 localStorage 기반 이력 조회
- **랜딩 페이지:** 싱글 페이지, 7개 섹션

### 제외 (Phase 2 이후)

- 회원가입/로그인
- 서버 DB (사용자 데이터 저장)
- 피부양자 신고, 보수월액 변경 등 추가 서식
- 외부 HR 시스템 API 연동 (Flex, Shiftee 등)
- 발송 결과 Webhook 수신
- 복합기 직접 연동 (데스크톱 에이전트)

## 랜딩 페이지 구성

싱글 페이지, 스크롤 기반 7개 섹션:

1. **히어로** — 프로덕트명, 카피, CTA "무료로 시작하기"
2. **페인포인트 공감** — "혹시 이런 경험 있으신가요?" (수기작성, 필드 혼란, 기한 불안)
3. **해결책 (How it works)** — 3단계 플로우 시각화 (입력 → 생성 → 발송)
4. **가격 비교** — 세무사 대행 vs ERP vs 신고해줘
5. **보안 강조** — "개인정보를 서버에 저장하지 않습니다"
6. **FAQ** — 지원 서식, 발송 확인, 복합기 이용 안내
7. **CTA** — "지금 바로 첫 신고를 시작하세요"

## 관할지사 매핑

데이터 소스: 4대보험 정보연계센터 지사찾기 (https://www.4insure.or.kr/pbiz/ntcn/selectInstBrofSrchView.do)

구조:
- 검색 키: 시/도 + 군/구 (사업장 주소 기반)
- 결과: 보험별(국민연금/건강보험/고용보험/산재보험) 관할 지사명, 주소, FAX번호
- 저장: JSON 파일로 정적 DB 구축 (전국 약 250개 시/군/구)
